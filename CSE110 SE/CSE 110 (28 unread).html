How to give us your application to test

Below is a summary of methods that have worked well previously that I wrote up last quarter. Please let us know if you have any questions.

 

If your team is writing the final project in Android, iOS, or as a web application this post may be of interest to you.

 

Last quarters of CSE 110, there were a lot of issues regarding turning in the projects for grading, so we thought making a guide on how to best turn in your final project at the end of week 10. First off, here are some reasons why you should care:

 

  1. It will make testing much simpler, which believe me is as much a benefit to you as it is to your grader.
  2. It will resolve compatibility issues between device and OS versions.
  3. You can avoid getting a phone call and losing 10 points over basic installation.

 

For iOS Projects:

For iOS apps it is easier to produce .IPA files that could be installed to an iPhone or iPad for testing. Please ensure that your application could support older iPhones at least up to iPhone 6 and iOS 9. This will help compatibility in general.

 

Creating IPA files in Xcode 10:

If you are developing on other versions of Xcode there are similar procedures to achieve the same goal.

 

Go to Window -> Organizer. Then select your app archive from archives. Then click the “Distribute App” button on the right panel.

 

 

Then follow the steps below

 

 

 

 

 

 

Then select the location you want to save your IPA file and click on export button. 

 

How to provide .ipa file:

In order to provide the IPA file to us you can put the file in google drive or dropbox and make the link public. Then submit the link that we can access to it in your README file as well as including in the google form that you will be submitting your project. (Test it to make sure we can download the file without any issue)

 

Test it before giving us access:

Before giving us access to the .ipa file make sure you install it on a clean device using one of many methods that I will include one below to know it actually works, so you won’t loose points. If we can’t test your app we can’t give points for the execution section of your project.

 

How to Install .ipa Files on Your iOS Device:

There are several ways but I include one here. Please include a detail instructions in the google form and README for us on how to install your app so we can follow the steps without calling you.

 

You need to have iTunes installed on your machine

Syncios app is installed on your machine: https://www.syncios.com/

 

  1. Open SYNCIOS app
  2. Connect iOS device to your machine
  3. Click on the application menu at the left
  4. Select the .ipa file from your computer
  5. Click on install button
  6. Wait for installing to complete
  7. On iOS device tap on settings -> General -> Profile and Device management
  8. Tap on “APPLICATION_NAME” below “Enterprise app”
  9. Tap on trust button
  10. Go to home and open the application and use it.

 

 

Other ways to distribute your app:

There are several other ways you could achieve this if your choose to like using TestFlight and even deploying to app store how ever you will need apple connect account and have developer accounts with them. If you choose to deploy to App Store or use TestFlight please give us detail instructions on how to access and download your app.

 

Instruction on how to setup TestFlight:

https://medium.com/@abhimuralidharan/testflight-public-beta-invite-links-ios-beta-testing-made-simple-97d2023791d0

 

For Android Applications:

 

SDK Version

First off, it's best to support as wide a range of Android versions as possible. This will help compatibility in general, but especially using this method. When you setup your project, there will be a dialog asking you to set the minimum SDK version.

 

I personally chose API 21 (Lollipop), since it has the widest support. Don't worry if you set the minimum version too high during setup; this can be changed anytime by changing build.gradle (Module: app).

 

Build APK

Once your project is working and ready to go, you can easily build an APK using Android Studio. Just click Build -> Build APK(s).

 

Once it's done, a message should pop up giving you the option to locate the generated APK, so you can find where it's saved on your system.

 

An APK is just a packaged, distributable version of your app. Now that you have this, you won't need to build using Android Studio in order to install your application.

 

Install Using ADB

By far the easiest way to install an APK is to use ADB from the command line. This works on both emulators and physical devices. (Note: if you use a physical device, USB debugging needs to be enabled and proper drivers need to be installed). From Android Studio, select the Terminal tab at the bottom, type the following command:

adb install path/to/apk

 

You may not have adb on your PATH (like me), so you may have to type the fully qualified path to adb.

 

Install Using Other Method

If you are using a physical device and you cannot get it to connect to ADB, worry not. As long as you can put the APK file on your device, you can install it. I personally like to use Google Drive, but any method will work. First make sure that the app you are using has "Install unknown apps" permissions. From the Settings App, click on Apps & notifications -> Advanced -> Special app access -> Install unknown apps (Note: this may vary across devices).

 

Now, all you have to do is open the APK file with Package installer, and you're all set!

 

 

For Web Applications:

 

If you want to use CSE 110 AWS credits to deploy your app for free, review the AWS part of the Docker and AWS Lab (Lab 7) for detailed instructions on creating an EC2 instance on AWS and accessing it.

 

If you do not want to use CSE 110 AWS credits, here are brief instructions:

Deploying the app on AWS:

You can use your AWS accounts given to your team to deploy the application and give us the url to test your application. I personally like this method but you could choose to host your application on your own web services and even buy a domain for it.

 

Login to your AWS account and head to EC2 console. Click on the launch button and select the Amazon Linux 2 AMI.

 

In the next page select t2.micro tier which is available for free. Then go to tab 6 to Configure Security Group at the top of the page. Security groups filter the traffic into and out of our instance — basically, who can access our virtual computer.

 

Security Groups:

You (and only you) will need to access the instance via ssh, so add a rule that allows Source “My IP” for SSH. We want others to be able to access our app in a web browser, so add a rule to allow HTTP access for all sources. 

 

Connecting to Server via SSH:

Once the instance is up and running, select it on the EC2 Instance dashboard (Services > EC2 > Running Instances) and hit Connect. This will give us the exact commands to connect to the instance.

 

Copy the example code, and paste it into Bash or a command prompt running in the folder with your private key (you generate this when launching your instance). Assuming everything goes well, you’ll be logged into your instance and see a familiar terminal command prompt.

 

Installing Requirements:

Install your app using GitHub and install all the requirements using sudo apt-get or curl. 

 

Running and Accessing the Web Application:

Run the app the same way you would run on your machine you could see the ip and the port that the app is running in the terminal.

While it looks like the the app is running on localhost:80/, that’s on the virtual machine. For us to access the web app, we’ll have to use the instance’s Public DNS IPv4 which can be found on the running instance dashboard.

 

 

Copy and paste the address into your browser, and you should see the application!

 

Provide us with that url in your REAMDE and the google form and make sure it doesn’t crash and we can access it on the day of grading. Test it as much as possible so it is not prone to errors and crashes. 

 

 

Concluding Remarks

We hope you find this guide useful when it comes time to turn in your projects. We highly recommend testing various methods in this guide and find the one that seems most natural to you. The more confident you are in the method, and the more thoroughly you test it with your specific application, the more likely your grader will have success installing on their end.

 

Finally, when you write your README that will contain the installation instructions for your app, make sure it is written by you and tailored to your application (e.g. using specific filenames, or screenshots of your own environment). Do not just copy and paste this guide, as it is not specific to your application and may result in ambiguous or incorrect instructions.

 

Also remember that we still need to access your source code to grade the code itself so don’t forget to give us access to your code as well.

 

Good luck!
