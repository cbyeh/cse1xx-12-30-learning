Unit 1: Introduction to Android

  • What is Android?

      Android is an operating system offered by Google. It runs in several different devices, such as phones, tablets, wearables, automobiles, TVs, etc. It uses Java under the hood, with many APIs available. An application programming interface (API) is a library provided by other developers to facilitate the use of certain generic functionalities. If you want to learn more about APIs, click here.

      Google is constantly releasing updates to Android. The (current) latest version of Android is called Android 10. If you want to learn more about Android 10 and its new features, click here. In this lab, you will get a brief introduction to some of the Android features and learn how to build a simple mobile app. This lab is NOT a comprehensive tutorial; it is, however, a way to get you acquainted with this platform, and hopefully inspire you to many great projects and products. Be bold, limitless, and don't be afraid of exploring what Android has to offer.

  • Why Android?
      In CSE110, you will have 10 weeks (in reality, ~8 weeks) to develop a working application from the ground up, and Android offers a great platform across different devices, a vibrant community, and regular updates announced every year during the Google I/O event.

In order to get the In Progress Check, you will need to run the First Application successfully on the day of the lab during your specific lab reservation time.

Unit 3: Creating an Android Virtual Machine

    Now, we will create an Android emulator to run our First Application. Emulators help test certain aspects of your application in different devices with relative ease. Keep in mind, though, that the emulator does not replace a real device, and it does NOT contain virtual hardware for some features such as Bluetooth, NFC, or Wi-Fi (< API 25). For this lab, the emulator will work just fine.
    NOTE: In general, it is preferred to test your Android apps using a real device instead of the emulator.

  • To get started, right click on your working area and click on Run 'app' . Alternatively, you can also click on the green Play button.
  • You will now be prompted to select a device or an emulator. Click on Create New Virtual Device to add a new emulator.
  • On the next screen, select Phone as the category, then select a device. Keep in mind that newer devices usually require more disk space. Click Next.
  • Now, download and select Pie (API 28) as the image to load onto the emulator. Click Next.
  • Finally, you will be able to review your selections. Click Finish when done.
  • You will, then, be brought back to select a device or an emulator. Your newly created emulator will be listed. Simply choose the emulator, and click OK.

  • If everything goes OK, the emulator will appear in a new window and load your app automatically.

    • If you get a message that SystemUI has stopped working, choose the option to wait. Do not close the application.
    • If you aren't taken directly to your app, "swipe" (drag your mouse) up on the Android home screen and select your FirstApplication app from the apps drawer that comes up.

    NOTE: Linux and Windows users might have some hardware acceleration issues. If so, please click here for help or check with a tutor.

4.3: Adding the Logic

  • Open MainActivity to link the functionality of the calculator to the simple design you implemented above.
  • Start by deleting the highlighted code shown below. This code implements the FloatingActionButton you deleted from your activity_main.xml.
  • When you generated your blank activity, Android Studio created a lot of starter code for you. One of these methods is onCreate method; this method is called when the activity or application starts up. This is what links your Design code or View from your functional code or Controller
    • This process starts with the line setContentView(R.layout.activity_main);
      This will link the application's MainActivity to the application's activity_main and content_main.
    • Most mobile app development follows the Model-View-Controller (MVC) architecture pattern or the Model-View-ViewModel (MVVM) architecture pattern. You can read more about them here and here
  • Create the following fields above the onCreate method. These fields are the instances of the objects that help you to control your application's layout.
    private EditText operand1;
    private EditText operand2;
    private Button btnAddition;
    private Button btnSubtraction;
    private Button btnDivision;
    private Button btnMultiplication;
    private TextView txtResult;
    private Button btnClear;

    If there are any errors like the one below, you can click on the line where error is occurring and use alt+ctrl (option + enter on Mac) to import the necessary library for any of the objects you created.

  • Under setContentView(R.layout.activity_main); add the following lines of code. Each of these lines will link an element from the content_main.xml to one of the fields that you just created above. This is achieved by using the findViewById() function which you can use to look up ids you assigned inside content_main.xml
    operand1 = (EditText) findViewById(R.id.input1);
    operand2 = (EditText) findViewById(R.id.input2);
    btnAddition = (Button) findViewById(R.id.plus);
    btnSubtraction = (Button) findViewById(R.id.minus);
    btnDivision = (Button) findViewById(R.id.divide);
    btnMultiplication = (Button) findViewById(R.id.multiply);
    txtResult = (TextView) findViewById(R.id.result);
    btnClear = (Button) findViewById(R.id.clear); If there are any issues, make sure your components' ID fields are what you expect them to be.

  • Now, you need to set up listeners for the buttons you created for the View of your application. Listeners will take actions from users and connect the requests to the controller of your application, so the application can perform the requested function. For this application you will use simple onClickListeners. They detect when one of the buttons is clicked by the user. Android has a variety of built in listeners for things such as swipes, taps, and more. The "listener" or "observer" pattern is the most common strategy for creating asynchronous callbacks within Android development. Listeners are used for any type of asynchronous event in order to implement the code to run when an event occurs. You could read more about listeners here.

  • You need to setup the setOnClicker like shown below. setOnClickListener is a method of the Button object in Android. This generates a new View and Override its onClick method.
  • When the button is clicked, the code inside onClick is run which, in this case, takes the input the user has placed in both input1 and input2 TextViews and stores them as doubles. The addition operation is performed, and the result TextView is updated to display the results to the user. Otherwise, if the inputs are not valid, a popup prompting the user to enter a valid input is displayed.
    • Place the following code at the bottom of the onCreate() method. 
    btnAddition.setOnClickListener(new View.OnClickListener(){
	@Override
	public void onClick(View v){
		if ((operand1.getText().length() > 0) && (operand2.getText().length() > 0 )){
			double oper1 = Double.parseDouble(operand1.getText().toString());
			double oper2 = Double.parseDouble(operand2.getText().toString());
			double theResult = oper1 + oper2;
			txtResult.setText(Double.toString(theResult));
		}
		else{
			Toast.makeText(MainActivity.this, "Please enter number in both operand fields", Toast.LENGTH_LONG).show();
		}
	}
});
  • Now implement the subtraction, multiplication, and division functionalities of your application following similar process as addition that you followed above. Ensure to name each btn according to its functionality. For example for division the button is called btnDivision.

    NOTE: The code is intentionally repetitive. Feel free to come up with a DRY (don't repeat yourself) solution yourself.

  • At last, you need to setup the clear button which will reset the inputs and the result display. Add the following code after your button code that you added in the onCreate() method. 
    
btnClear.setOnClickListener(new View.OnClickListener(){
	@Override
	public void onClick(View v){
		operand1.setText("");
		operand2.setText("");
		txtResult.setText("(result will appear here)");
		operand1.requestFocus();
	}
});
  • Now test your application. In the upper right hand corner click on the play button.
    • A window like the one below will pop up. Choose Launch emulator and click OK to open your Android VM or load your Android device.
    • If there is no emulator found, use the Create Virtual Device button to create one. Use all of the default settings.
    • Once the emulator is launched (this may take a while), you should see something similar to the following.
    • If you aren't taken directly to your app, "swipe" (drag your mouse) up on the Android home screen and select your Calculator app from the apps drawer that comes up.
